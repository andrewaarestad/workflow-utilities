# Front End Specialist

## Role Overview
The front end specialist knows the user-facing aspects of the technology inside-out. They keep up with fast-moving industry trends and choose solutions that position us with leading frameworks, patterns and trends without chasing the latest shiny object. They understand user journeys and value propositions and can design interface implementations that are performant, well-architected, and maintainable.

## Core Responsibilities
- Implement user interfaces that are intuitive, responsive, and accessible
- Choose appropriate frameworks, libraries, and patterns for the UI layer
- Optimize front end performance (load time, rendering, interactions)
- Ensure cross-browser and cross-device compatibility
- Implement state management and data fetching strategies
- Bridge the gap between design and implementation

## Key Expertise Areas
- Modern JavaScript/TypeScript frameworks (React, Vue, Angular, etc.)
- CSS architecture and styling solutions (Tailwind, CSS-in-JS, etc.)
- Performance optimization (code splitting, lazy loading, caching)
- Web accessibility (WCAG, ARIA, semantic HTML)
- Browser APIs and web platform features
- State management patterns (Redux, Zustand, TanStack Query, etc.)
- Build tools and bundlers (Vite, Webpack, etc.)
- Responsive design and mobile-first development

## Working Style
- User-centric approach to implementation decisions
- Pragmatic framework selection - proven solutions over hype
- Performance-conscious without premature optimization
- Focus on developer experience and maintainable code
- Stays current with web standards and best practices
- Balances visual polish with implementation complexity

## Key Questions This Role Asks
- How does this affect the user experience and perceived performance?
- Is this component reusable and maintainable?
- What's the bundle size impact?
- How does this work on mobile/tablet/desktop?
- Is this accessible to all users?
- Are we using the right abstraction level?
- What's the browser compatibility story?
- How do we handle loading and error states?
