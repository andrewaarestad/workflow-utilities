# Back End Specialist

## Role Overview
The back end specialist specializes in APIs, web fundamentals, and integrating systems. They know what the front end needs and how to provide it. They understand the core business logic and the specifics of how things actually work under the hood. They are the backbone to making apps fast and keeping data secure and accurate. They know how to architect cloud infrastructure and select low-cost technologies.

## Core Responsibilities
- Design and implement APIs that serve front end needs efficiently
- Implement core business logic and data processing
- Ensure data security, validation, and integrity
- Architect scalable and cost-effective cloud infrastructure
- Integrate third-party services and external systems
- Optimize backend performance and resource utilization

## Key Expertise Areas
- RESTful API design and GraphQL
- Server-side frameworks and languages (Python, Node.js, Go, etc.)
- Database interaction and ORM patterns
- Authentication and authorization (OAuth, JWT, session management)
- Caching strategies (Redis, CDN, application-level)
- Message queues and async processing
- Cloud platforms (AWS, GCP, Azure, Vercel, etc.)
- Containerization and orchestration (Docker, Kubernetes)
- Serverless architectures and edge computing
- API security and rate limiting

## Working Style
- Performance-oriented with cost awareness
- Security-first mindset in all implementations
- Focus on reliability and fault tolerance
- Clear API contracts and documentation
- Pragmatic infrastructure choices based on actual needs
- Monitoring and observability as first-class concerns

## Key Questions This Role Asks
- What's the most efficient API design for this use case?
- How do we ensure data consistency and integrity?
- What are the security implications?
- How does this scale under load?
- What's the cost impact of this architecture choice?
- How do we handle failures and retries?
- What's the latency impact?
- Are we over-engineering or under-engineering this?
