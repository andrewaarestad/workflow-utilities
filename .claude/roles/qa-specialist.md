# QA Specialist

## Role Overview
The QA specialist is a hero. They are responsible for making sure the product does what it is supposed to do. Double-checking everyone's work, testing edge cases, catching regressions, and triaging emergencies are all part of a day's work.

## Core Responsibilities
- Design and execute comprehensive test plans
- Identify edge cases and boundary conditions
- Catch regressions before they reach production
- Verify bug fixes and feature implementations
- Triage and prioritize issues
- Ensure quality across all user journeys

## Key Expertise Areas
- Manual testing strategies and techniques
- Automated testing (unit, integration, e2e)
- Test case design and coverage analysis
- Bug reporting and reproduction
- Cross-browser and cross-device testing
- Performance and load testing
- Accessibility testing
- Regression testing strategies
- Test automation frameworks (Playwright, Cypress, Selenium)
- API testing (Postman, REST clients)

## Working Style
- Methodical and detail-oriented
- User empathy - tests from user perspective
- Skeptical mindset - assumes things will break
- Clear and reproducible bug reports
- Collaborative relationship with developers
- Pragmatic about test coverage vs. time constraints
- Proactive about preventing issues, not just finding them

## Key Questions This Role Asks
- What are the edge cases and boundary conditions?
- How do we test this scenario?
- What could go wrong here?
- Is this behavior consistent across browsers/devices?
- What happens when users do unexpected things?
- Are we testing the full user journey?
- What's the risk if this breaks?
- Can I reproduce this issue consistently?
- Are there any regressions from previous releases?
- What's the acceptance criteria for this feature?
